# 首页看板采购模块扩展设计

## 概述

在现有首页看板新增采购概览 KPI 卡片，与客户统计 KPI 并排展示。采购模块不做独立图表，以 4 项核心指标卡形式补充。

## 设计决策

- **定位**：销售 + 产值为主角，采购作为 KPI 指标补充（用户确认）
- **布局**：方案 B — 客户 KPI 与采购 KPI 并排一行，图表下移一行
- **交互**：纯展示，无点击跳转
- **数据**：静态 mock 数据，后续可替换为 API

## 布局结构

```
┌──────────────────────────────────────────────┐
│  Row 1: 年销售情况（折线图 · 全宽）            │
├──────────────────┬───────────────────────────┤
│  Row 2 左:       │  Row 2 右:                │
│  新增客户情况统计  │  采购概览（新增）           │
│  (4个KPI数字卡片) │  (4个KPI数字卡片)          │
├──────────────────┼───────────────────────────┤
│  Row 3 左:       │  Row 3 右:                │
│  月度产值（柱状图）│  销售额排名（横向柱状图）   │
└──────────────────┴───────────────────────────┘
```

## 采购指标

| 指标 | 字段名 | mock 值 | 说明 |
|------|--------|---------|------|
| 本月采购金额 | `monthlyAmount` | ¥8,520万 | 金额格式与客户统计一致 |
| 采购成本环比 | `costChange` | -3.2% | 负值为降，绿色显示 |
| 在途订单数 | `inTransitOrders` | 47 | 纯数字 |
| 库存周转天数 | `inventoryDays` | 28 | 纯数字 |

## 数据层

在 `src/data/dashboard.js` 新增 `purchaseStats` 字段，与现有 `customerStats` 结构同级：

```js
purchaseStats: {
  monthlyAmount: '¥8,520万',
  costChange: '-3.2%',
  inTransitOrders: 47,
  inventoryDays: 28
}
```

## 组件层

`HomePage.vue` 新增一个 `<el-card>` 采购概览卡片，完全复用现有 KPI 样式类（`.kpi-grid` / `.kpi-item` / `.kpi-num` / `.kpi-label`）。`<script>` 部分零改动，CSS 零新增。

卡片顺序调整：
1. 年销售情况（折线图 · `.widget-full`）
2. 新增客户情况统计（KPI 卡片）
3. 采购概览（KPI 卡片 · 新增）
4. 月度产值（柱状图）
5. 销售额排名（横向柱状图，移除 `.widget-full`）

## 技术约束

- 2 列网格由 `.widget-grid { grid-template-columns: 1fr 1fr }` 控制
- 只有「年销售情况」卡片使用 `.widget-full { grid-column: 1 / -1 }`
- 其余 4 个卡片自动填入 2 列流式布局
